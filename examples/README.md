# Examples

This folder will contain representative output from real builds of this skill — sanitized to remove organization-specific details — so you can see what a finished context library actually looks like before running the skill yourself.

Planned examples:

- **Foundation module (F0).** A worked example of a behavioral-standards module showing the gate structure, runtime perspective, and metaprompting tone.
- **Agent definition with manifest.** An agent file with `always_load:` / `conditional:` classification and `load_when:` triggers demonstrating Trigger Discipline.
- **Proposal — Ownership and Use-Shape table.** The Phase 3 deliverable that commits each module to one of four use-shapes (cross-reference, subset, invocation by name, reach-beyond).
- **Phase 2 comprehension artifacts.** Per-source notes, signal log, expectations-vs-findings, and convergences — illustrating the recognition/synthesis split.

Examples are added when a build produces artifacts general enough to publish without sanitization concerns. Until then, the [Make Good context library](https://github.com/WeMakeGood/makegood-context-library) is itself a finished build of this skill and can serve as a reference.
